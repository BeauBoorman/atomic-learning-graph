# Using the course exports

Atomic Learning has one authority: the committed `data/graph.json`. The files below are deliberate,
format-specific projections of that graph, not a second place to edit course content or relations.
`pnpm gate` checks the graph-derived bytes for the supported exports; it does **not** automate every
third-party application's import UI. Open the format in the app you use and inspect it before relying
on it for a course.

## Obsidian

[`exports/obsidian/`](../exports/obsidian/) is a ready-to-open vault: one Markdown note per concept,
with a full cited lesson and linked prerequisites. Clone the repository or download its ZIP, then in
Obsidian choose **Open folder as vault**, select `exports/obsidian`, and begin with
[`Start Here.md`](../exports/obsidian/Start%20Here.md). The vault is regenerated by
`pnpm emit:obsidian` and checked by `pnpm verify:obsidian`.

## org-roam

[`atomic-learning-graph.org`](../atomic-learning-graph.org) is the course as one Org file with
org-roam IDs and a prerequisite-ordered **Learning Path**. Put the file in your org-roam directory,
open it in Emacs, run `M-x org-roam-db-sync`, then start at **Learning Path**. It is regenerated by
`pnpm emit:orgroam` and checked by `pnpm verify:orgroam`.

## Anki

[`atomic-learning-graph-anki.tsv`](../atomic-learning-graph-anki.tsv) is a tab-separated import with
Basic notes, a named deck, tags, question fronts, and source receipts on answer backs. In Anki choose
**File → Import**, select the TSV, keep the **Basic** note type, confirm the deck/tag settings from
the file header, and import. It is regenerated by `pnpm emit:anki` and checked by
`pnpm verify:anki`.

## Tinderbox

[`atomic-learning-graph.opml`](../atomic-learning-graph.opml) is the supported, deterministic
interchange artifact. In Tinderbox choose **File → Import…**, select the OPML, open **Concepts** in
Map view, and follow the connected prerequisite cards. The import applies the map's hierarchy,
metadata, prototypes, colors, and positions. It is regenerated by `pnpm emit:tinderbox` and checked
by `pnpm verify:tinderbox`.

[`atomic-learning-graph.tbx`](../atomic-learning-graph.tbx) opens directly as a hand-finished
convenience file. It is not rebuilt or byte-checked by the pipeline, so use the OPML when you need
the supported reproducible interchange path.

## Logseq

There is no first-class Logseq course export yet. The closest portable material is the Markdown
course in [`exports/obsidian/`](../exports/obsidian/): copy those notes into a local Logseq graph and
open **Start Here**. The notes use Obsidian-style wiki links, so check the resulting links and page
layout in your Logseq setup before relying on it. This is a manual compatibility route, not a
Logseq-specific artifact or a `pnpm gate` guarantee.

For a small, app-neutral product tour rather than the course itself, the separate
[`exports/showcase/markdown/`](../exports/showcase/markdown/) guide is also plain Markdown.
