# Provenance patches — disclosed hand-edits to `data/graph.json`

> **Honesty disclosure.** `data/graph.json` is supposed to be emitted only by `pnpm atomize`
> and byte-pinned to a recorded run (see `docs/adr/001-commit-the-generated-graph.md`).
> This file lists every hand-patch that has ever been applied to it outside that path, so the
> project's "machine-emitted, never hand-edited" claim has an honest exception list instead of
> a silent one. The same records are embedded in `data/graph.run.json` under `humanPatches`
> for machine readability; this Markdown file exists so a reviewer finds the disclosure by
> browsing the repo root.

## How to read this file

- Each entry corresponds to one commit that hand-edited `data/graph.json`.
- The entry names the exact field touched, what changed and why, what was **not** touched
  (citations and structure invariants in particular), and how the pinned hashes were
  brought back into lockstep with the patched bytes.
- If this file is empty apart from this header, there are no disclosed hand-patches on record.

## 2026-07-20 · commit `f65a0fb` · self-attention step-3 worked example

**Field touched:** `data/graph.json` → `concepts[self-attention].lesson.steps[2]` (the third
teaching step of the `self-attention` concept).

**What changed:** The worked example for that step originally averaged all token values
uniformly — `(2 + 4 + 6) ÷ 3 = 4`, producing the output `4, 4, 4`. That is *average pooling*,
the degenerate equal-weights case, not attention. The example was rewritten to use
position-dependent attention weights (e.g. `0.5 / 0.3 / 0.2`), so the output shows each token
attending to the others to a different degree — the defining property of attention. The same
rewrite was carried through the step's six passion analogies so they remain faithful to the
new arithmetic.

**Why:** The original example was flagged by
[`plans/reports/opencode-v7-quality-review.md`](plans/reports/opencode-v7-quality-review.md)
as mathematically misleading: a beginner who read it would conclude that self-attention
"blurs everything to the mean," which is the opposite of what attention does. The fix replaces
a falsehood with an accurate illustration of the actual mechanism.

**What was NOT touched:**
- `concept.provenance.quotedText` — the verbatim citation that grounds the step.
- `concept.lesson.steps[*].citation` on every step of every concept.
- `concept.id`, `concept.title`, `concept.summary`, `concept.tags`.
- `edges` — the nine-edge prerequisite spine is unchanged.
- Every other concept in the graph.

The patch touches **lesson prose and its analogies only** — the parts the project already
labels as "model-generated, advisory content." No deterministic invariant (provenance,
license, structure, path) was altered.

**Hash restamp:** `graphSha256` in `data/graph.run.json` was recomputed to match the patched
graph bytes. `data/course.receipt.json` was regenerated by `pnpm emit:receipt` so its
`verification.graphHash` continues to mirror `graph.run.json`. The deterministic gates in
`src/atomization/graph-run.test.ts` and `src/atomization/course-receipt.test.ts` continue to
assert the post-patch hashes against the post-patch bytes; nothing was loosened to make them
pass.

**Single exception:** This is the **only** hand-patch to `data/graph.json` on record as of
this commit. The graph remains byte-reproducible only through `pnpm atomize`; this patch is
the disclosed exception, not a new pattern. Any future hand-patch must be recorded here and
in `data/graph.run.json.humanPatches` before it ships.
