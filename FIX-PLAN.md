## Step 3 — Regenerate stale exports (cause c)

The gate runs, in `scripts/gate.sh`, a `verify:*` stage per format that byte-compares committed
artifacts to freshly-emitted bytes:
`verify:llms, verify:orgroam, verify:tinderbox, verify:anki, verify:obsidian, verify:exam,
verify:receipt, verify:showcase`. Because the emitters now strip control chars (and other edits),
committed bytes are stale. Regenerate every artifact deterministically. **These emitters make no
model/API calls** (they read `data/graph.json` + write files), so this costs **$0** and is safe to
run in full.

- **Commands (exact):**
```
pnpm emit:llms
pnpm emit:orgroam
pnpm emit:tinderbox
pnpm emit:anki
pnpm emit:obsidian
pnpm emit:exam
pnpm emit:showcase
pnpm emit:receipt
```

Expected effect: the `softmax-ordering` U+0003 control byte is stripped from every export (the
intended Focus-2c behavior). The regenerated bytes must be **committed** as part of the fix — they
are the artifacts the gate checks.
